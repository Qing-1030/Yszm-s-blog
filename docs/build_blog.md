# 我是如何用 Python 搭建个人博客的

这是我博客的第一篇正式技术文章。在这篇文章里，我想分享我是如何利用 Python 和开源工具，在零成本的情况下搭建起这个独立博客的。

## 为什么选择“折腾”？

其实现在的写作平台很多（知乎、公众号等），但我依然选择自己搭建一个博客，原因很简单：

1. **数据自主**：所有的文章源码都保存在我自己的 GitHub 仓库里，不会因为平台规则变动而丢失。
2. **极客精神**：作为一名编程学习者，用代码构建自己的“后花园”本身就是一件很有成就感的事。
3. **技术积累**：在搭建过程中，我顺便熟悉了 Python 环境、命令行操作和 Git 版本控制。

## 我的技术栈

我的博客并不是从零开始写 HTML/CSS，而是站在了巨人的肩膀上。以下是我使用的核心工具：

| 工具 | 作用      | 备注 |
| :--- |:--------| :--- |
| **Python** | 基础语言环境  | 强大的生态支持 |
| **MkDocs** | 静态网站生成器 | 把 Markdown 转换成网页的引擎 |
| **Material** | 主题皮肤    | 提供美观的 UI 和响应式设计 |
| **GitHub Pages** | 网站托管    | 免费的云端服务器 |
| **PyCharm** | 我的开发神器  | 编写和管理代码的地方 |

## 搭建过程复盘

整个过程比我想象中要简单，核心只有三步：

### 1. 安装“引擎”
利用 Python 的包管理工具 pip，安装 MkDocs 和 Material 主题：

```bash
pip install mkdocs-material
```

### 2. 初始化项目

一行命令生成博客的基础骨架：

```bash
mkdocs new .
```

### 3. 配置与发布

在 `mkdocs.yml` 中配置好网站名称和主题后，使用 MkDocs 提供的命令一键部署到 GitHub：

```bash
mkdocs gh-deploy
```

就是这么简单！剩下的工作，就是专注于用 Markdown 写作了。

## 遇到的坑与解决

在搭建过程中，我也遇到了一些小插曲（比如 Markdown 代码块的语法问题），但通过查阅文档和调试，最终都解决了。这也让我明白：**编程不仅仅是写代码，更是解决问题的过程。**
